# Аналитический инструмент для металлургических данных

Научно-прикладной прототип для анализа производственных данных с выявлением зависимостей между технологическими параметрами и КПЭ.

## Цель проекта

Создание инструмента для:
- Автоматического анализа металлургических данных
- Выявления скрытых зависимостей в технологических процессах  
- Кластеризации режимов работы оборудования
- Интерпретации результатов через понятные правила

## Использование системы

### Поддерживаемые форматы данных
- **CSV файлы** - разделитель запятая
- **Excel файлы** - .xlsx формат
- **Размер данных**: от 100 до 100,000 записей
- **Типы данных**: числовые и категориальные признаки

### Этапы анализа

1. **Загрузка данных** 
   - Импорт CSV/Excel файлов
   - Автоматический анализ структуры данных
   - Превью и проверка качества

2. **Выбор признаков**
   - Отбор релевантных параметров
   - Разделение на числовые и категориальные
   - Исключение неинформативных столбцов

3. **Фильтрация данных**
   - Удаление выбросов (IQR метод)
   - Ограничение диапазонов значений
   - Очистка от аномальных значений

4. **Предобработка**
   - Автоматическая нормализация (Standard/MinMax)
   - Кодирование категорий (OneHot/Label Encoding)
   - Обработка редких категорий

5. **Кластеризация**
   - Выбор алгоритма кластеризации
   - Автоматическое определение числа кластеров
   - Вычисление метрик качества

6. **Визуализация**
   - PCA проекции кластеров
   - Дендрограммы
   - Распределения признаков
   - Интерактивные графики

7. **Интерпретация**
   - Построение дерева решений
   - Важность признаков
   - Правила разделения кластеров

8. **Экспорт результатов**
   - CSV с присвоенными кластерами
   - JSON отчет с метриками
   - Визуализации в высоком разрешении

### Алгоритмы кластеризации

| Алгоритм | Описание | Когда использовать |
|----------|----------|-------------------|
| **Agglomerative** | Иерархическая кластеризация | Небольшие данные (<5k), нужна дендрограмма |
| **DBSCAN** | Кластеризация по плотности | Данные с выбросами, неизвестное число кластеров |
| **FAISS K-means** | Оптимизированный K-means | Большие данные (>10k записей) |

### Метрики качества

- **Silhouette Score** (0.7-1.0 отлично, 0.5-0.7 хорошо)
  - Качество разделения кластеров
  - Чем выше, тем лучше

- **Davies-Bouldin Index** (<1.0 отлично, 1.0-2.0 хорошо)  
  - Компактность кластеров
  - Чем ниже, тем лучше

- **Calinski-Harabasz Index** (>1000 отлично)
  - Соотношение межкластерной и внутрикластерной дисперсии
  - Чем выше, тем лучше

## API и интеграция

### Программный интерфейс

```python
from src.preprocessing import MetallurgyPreprocessor
from src.clustering import MetallurgyClustering
from src.explain_tree import ClusterExplainer

# Полный пайплайн анализа
def analyze_metallurgy_data(df):
    # Предобработка
    preprocessor = MetallurgyPreprocessor()
    X = preprocessor.fit_transform(df)
    
    # Кластеризация
    clustering = MetallurgyClustering()
    clustering.fit_agglomerative(X)
    
    # Интерпретация
    explainer = ClusterExplainer()
    explainer.fit(df, clustering.labels_)
    
    return {
        'labels': clustering.labels_,
        'metrics': clustering.calculate_metrics(),
        'interpretation': explainer.generate_cluster_interpretation_report()
    }
```

## Поддержка и документация

### Часто задаваемые вопросы

**Q: Какой максимальный размер данных?**
A: До 100,000 записей. Для больших данных используйте FAISS K-means.

**Q: Поддерживается ли GPU ускорение?**
A: Да, FAISS может использовать GPU при установке faiss-gpu вместо faiss-cpu.

**Q: Как добавить свои метрики качества?**
A: Расширьте метод `calculate_metrics()` в классе `MetallurgyClustering`.
